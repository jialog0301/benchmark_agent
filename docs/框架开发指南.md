## 系统架构

```
User Input
    ↓
Query Planner (B) ──→ planner_result.json
    ↓
Search Agent (B) ──→ raw_report.md
    ↓
Benchmark Extractor (C) ──→ benchmarks.json
    ↓
Task-Fit Scorer (D) ──→ ranked_benchmarks.json
    ↓
LLM Judge (D) ──→ ranked_benchmarks.json (补充 recommendation_reason)
    ↓
Report Writer (E) ──→ final_report.md
    ↓
Streamlit UI (E)
```

### scorer.py 接口

```python
def score_benchmarks(benchmarks: list[dict], mode: str) -> list[dict]:
    """
    计算 Task-Fit Score 并排名

    Returns:
        [
            {
                **benchmark,  # 保留所有字段
                "mode": str,
                "task_fit_score": float,   # 保留两位小数
                "rank": int               # 1, 2, 3, ...
            },
            ...
        ]
    """
```

### Task-Fit Score 评分公式

| 模式 | 公式 | 权重 |
|------|------|------|
| **课程实验** | 0.30×Teaching + 0.25×Resource + 0.20×(6-ReproductionDiff) + 0.15×Popularity + 0.10×TimeCost | - |
| **科研调研** | 0.30×Research + 0.25×Authority + 0.20×Popularity + 0.15×Resource + 0.10×Coverage | Coverage 用 evaluated_ability 数量替代 |
| **快速复现** | 0.35×(6-ReproductionDiff) + 0.25×Resource + 0.20×Documentation + 0.15×TimeCost + 0.05×Popularity | - |

**注意**：`reproduction_difficulty` 越高越难，所以要 `6 - reproduction_difficulty` 转成友好度

## Prompt 模板使用方式

所有 prompt 模板位于 `prompts/` 目录，结合 `src/llm_client.py` 调用 LLM：

```python
from src.llm_client import call_llm

# 加载 prompt
with open("prompts/extractor.md", "r", encoding="utf-8") as f:
    template = f.read()

# 填充变量（Python format string）
prompt = template.format(topic=topic, raw_report=raw_report)

# 调用 LLM（自动使用 .env 中的配置）
response = call_llm(prompt, json_mode=True)

# 解析输出
result = json.loads(response)
```

## LLM 配置

### 环境变量设置

创建 `.env` 文件（从 `.env.example` 复制）：

```bash
cp .env.example .env
```

编辑 `.env`，填入你的 API 配置：

```bash
# MiniMax API (推荐，使用 Anthropic SDK)
MINIMAX_API_KEY=your_api_key_here
MINIMAX_BASE_URL=https://api.minimax.chat/v1
MINIMAX_MODEL=MiniMax-M2.7

# 或者使用 Anthropic 官方
# ANTHROPIC_API_KEY=your_api_key_here
# ANTHROPIC_BASE_URL=https://api.anthropic.com

# 或者使用本地模型 (OpenAI 兼容接口)
# OPENAI_API_KEY=dummy
# OPENAI_BASE_URL=http://localhost:11434/v1
```

### 调用方式

```python
from src.llm_client import call_llm

# 基础调用
response = call_llm("你好，请介绍一下自己")

# JSON 输出模式（返回 JSON 字符串）
response = call_llm(
    prompt='请以JSON格式输出 {"name": "张三"}',
    json_mode=True
)

# 带系统提示
response = call_llm(
    prompt="提取以下文本中的关键词",
    system="你是一个专业的文本分析助手",
    max_tokens=1000
)

# 指定模型
response = call_llm(
    prompt="写一段代码",
    model="MiniMax-M2.7",
    max_tokens=2000
)
```

### 模型优先级

1. 优先使用 `MINIMAX_API_KEY`（如有设置）
2. 次之使用 `ANTHROPIC_API_KEY`（如有设置）
3. 最后降级到 `OPENAI_BASE_URL`（用于 ollama 等本地模型）

### MiniMax-M2.7 特性

MiniMax-M2.7 支持 `thinking` 输出，会自动包含在返回结果中：

```python
response = call_llm("解释一下什么是机器学习")
# 返回可能包含:
# [Thinking]: 模型思考过程...
# 实际回答内容...
```
